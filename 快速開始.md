# ⚡️ 快速開始（5 分鐘）

## 1️⃣ 本地測試（同一電腦 - 無需 Firebase）

### 開啟遊戲
```
直接雙擊 index.html
或在瀏覽器輸入: file:///c:/Users/FionaCheng/Downloads/bingo_test/index.html
```

### 多人遊戲
```
1. 第 1 個分頁 → 輸入「玩家 A」→ 進入遊戲
2. 開新分頁 → 輸入 URL（同一個頁面）→ 輸入「玩家 B」
3. 玩家 A 版頁點擊 「🐎 抽球號」
4. 🎉 玩家 B 的棋盤會自動同步！
```

---

## 2️⃣ 跨設備測試（手機 + 電腦 - 需要 Firebase）

### 步驟 A：設定 Firebase（5 分鐘）

1. 訪問: https://console.firebase.google.com
2. 建立新專案 → 名稱: `Bingo Game`
3. 建立 Web 應用 → 複製配置
4. 建立 Realtime Database → 測試模式
5. 規則改成:
   ```json
   {
     "rules": {
       "games": {
         "$sessionId": {
           ".read": true,
           ".write": true
         }
       }
     }
   }
   ```

### 步驟 B：更新遊戲配置（3 分鐘）

用記事本打開 `index.html`，找到（約第 430 行）：

```javascript
const firebaseConfig = {
    apiKey: "AIzaSyDp_1234567890abcdefghijklmnopqrst",
    ...
};
```

**貼上 Firebase 控制台給你的配置**

### 步驟 C：架設本地伺服器（1 分鐘）

在 PowerShell 中執行:
```powershell
cd c:\Users\FionaCheng\Downloads\bingo_test
python -m http.server 8000
```

### 步驟 D：測試跨設備（2 分鐘）

**電腦端:**
```
1. 打開: http://localhost:8000
2. 輸入「玩家 A」→ 進入遊戲
3. 點擊「🐎 抽球號」
```

**手機端:**
```
1. 同 WiFi 網路下打開: http://192.168.1.X:8000
   （X 從 ipconfig 查詢）
2. 輸入「玩家 B」→ 進入遊戲
3. 看著手機棋盤自動同步！ 🎉
```

## 🎯 核心概念

### 會話 ID（Session ID）
```
遊戲會話 ID: session_1623456789_abc1234567

作用: 識別一局遊戲，所有玩家共享同一個 ID
```

### 自動同步流程
```
玩家 1 抽號 → LocalStorage/Firebase → 玩家 2 收到 → 自動標記 ✅
```

### 棋盤數據
```
每個玩家有自己的棋盤 (24 個號碼 + 1 個「發」中心)
但所有玩家共享已抽的號碼清單
```

## 🚨 常見錯誤排查

### ❌ 跨設備無法同步

| 症狀 | 檢查項目 |
|------|--------|
| 手機連不到電腦 | WiFi 不同？手機無法連 192.168.1.X？ |
| 號碼沒同步 | Python 伺服器有沒有運行？ |
| Firebase 錯誤 | 配置是否複製正確？API Key 有沒有虛假？ |

### 查看錯誤
```
1. 打開瀏覽器 F12
2. 選擇「Console」標籤
3. 看有沒有紅色的錯誤訊息
```

## 📱 快速分享遊戲

遊戲進行中會顯示：
```
📌 遊戲會話
session_1623456789_abc1234567
[📋 複製]
```

分享給其他玩家：
1. 點擊「📋 複製」複製會話 ID
2. 分享遊戲 URL + 會話 ID 給朋友
3. 朋友加入時會自動偵測並加入同一局
